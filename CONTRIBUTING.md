# Contributing to d3vsh0p

Thanks for your interest in d3vsh0p!

## Before You Start

**Please open an issue before submitting a pull request.** This lets us discuss the change, confirm it's something we want, and avoid wasted effort on both sides.

PRs submitted without a corresponding issue may be closed without review.

## What We're Looking For

- Bug fixes with a clear reproduction case
- Documentation improvements
- Test coverage improvements

## What We're Probably Not Looking For

- Large feature additions without prior discussion
- Refactors or style changes
- Dependency upgrades (we manage these internally)

## Process

1. Open an issue describing the bug or improvement
2. Wait for a maintainer to confirm the approach
3. Fork the repo and create a feature branch (`fix/description` or `feat/description`)
4. Make your changes, ensuring all tests pass (`cd platform/orchestrator && npm test`)
5. Submit a PR referencing the issue

## Code of Conduct

Be respectful. We're building in the open but this is a small project. Keep discussions constructive.
